使用Artalk代替不蒜子统计PV

Artalk统计API说明

此API可以在Artalk官方的Openapi中找到

全站Pv总数获取

POST /stat
传入参数Query 功能 值 必须
type 获取数据的类型 “latest_comments”， “latest_pages”， “pv_most_pages”， “comment_most_pages”， “page_pv” “site_pv”， “page_comment”， “site_comment” “rand_comments”， “rand_pages” 是
page_keys 页面唯一ID [String] 否
site_name 限定数据只在哪个站点获取 [String] 站点的名称 否
limit 限定最大获取数量 否

Type 参数解析

Type参数 功能
latest_comments 获取最新的评论数据，可以使用 limit 限定数量
latest_pages 获取最新页面，可以使用 limit 限定数量
pv_most_pages 获取访问(PV)最多的页面，可以使用 limit 限定数量
comment_most_pages 获取评论数最多的页面，可以使用 limit 限定数量
page_pv 获取某个页面的访问(PV)，需要传入page_keys
site_pv 获取全站的访问(PV)
page_comment 获取某个页面的评论数，需要传入page_keys
site_comment 获取全站的评论数
rand_comments 随机评论，可以使用 limit 限定数量
rand_pages 随机页面，可以使用 limit 限定数量

Responses返回值

data any 数据
extra any 数据
msg string 消息
success boolean 是否成功

Nuxt实现

<template>
  <div class="static_warp">
      ......
    <div class="static_item">
        <span class="static_span">
          访问
        </span>
        <span class="static_num">
          {{ pv.success ? pv.data : '无法获取' }}
        </span>
    </div>
    <div class="static_item">
        <span class="static_span">
          评论
        </span>
      <span class="static_num">
          {{ comments.success ? comments.data : '无法获取' }}
        </span>
    </div>
      ......
  </div>
</template>

<script setup lang="ts">
  /*
    侧栏统计组件
   */

  import { onMounted } from "#imports";
  const pv = useLazyAsyncData('pv', () => $fetch('https://你的Artalk域名/api/stat', {
    method: 'post',
    headers: {
      'origin': '你的源站'
    },
    query: {
      'type': 'site_pv',
      'site_name': 'Rootlex 的博客',
    },
  }).catch(() => {
    console.log('无法获取PV')
  })).data
  const comments = useLazyAsyncData('comments', () => $fetch('https://你的Artalk域名/api/stat', {
      method: 'post',
      headers: {
        'origin': '你的源站'
      },
      query: {
        'type': 'site_comment',
        'site_name': '你站点的名字',
      },
    }).catch(() => {
      console.log('无法获取总comments数')
    })).data

  onMounted(async() => {
    await $fetch('https://你的Artalk域名/api/pv', {
      method: 'post',
      query: {
        'page_key': '你的源站',
        'site_name': 'Rootlex 的博客',
      },
    }).catch(() => {
      console.log('无法提交PV')
    })
  })

</script>

在使用 useLazyAsyncData 需要传入 origin，因为服务端并不知道你的 origin 所以不会自动传入。如果我们不传入Artalk会阻止我们访问API。而在组件完成挂载后（onMounted）请求则不用传入。

这里我在组件挂载后请求了 /api/pv 这个接口，用来增加和统计首页的PV

原生JS实现

原生JS可以通过 promise.all 和 fetch 实现，此方法尚未经过博主测试，请谨慎使用，但JS写法应该是差不多的

const siteName = '站点名称'
onload(()=>{
    const [pv, comments, postpv] = Promise.all([
        fetch('https://你的Artalk域名/api/stat' + '?type=site_pv' + '?site_name=' + siteName, {
            method: 'post',
        }).catch(()=>{
            console.error('无法获取PV')
        }),
        fetch('https://你的Artalk域名/api/stat' + '?type=site_comment' + '?site_name=' + siteName, {
            method: 'post',
        }).catch(()=>{
            console.error('无法获取总comments数')
        }),
        fetch('https://你的Artalk域名/api/pv' + '?page_key=' + window.location, {
            method: 'post'
        }).catch(()=>{
            console.error('提交PV失败')
        }),
    ]).catch(()=>{
        console.error('error')
    })
    console.log(pv,comments)
})
